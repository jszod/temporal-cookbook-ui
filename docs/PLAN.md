## Usage Notes

### Update Frequency
- **Add feature**: When starting work (~5 seconds)
- **Update status**: As you make progress (~5 seconds)
- **Move to completed**: When feature merged (~5 seconds)

### Workflow Options

**With GitHub Issues:**
1. Create GitHub issue for feature
2. Add issue link to this file
3. Create feature PRD, link from issue body
4. Update status notes here as you work

**Without GitHub Issues:**
1. Add feature title directly (no link)
2. Create feature PRD
3. Track status here
4. Works great for prototypes and solo projects

**Hybrid:**
- Mix of linked and unlinked features
- Use GitHub when you want external visibility
- Keep local when prototyping

### Integration with Other Files

- **GitHub Issues** - External tracking, links to PRDs
- **Feature PRDs** (`docs/features/`) - Detailed specifications
- **Task Files** (`docs/tasks/`) - Implementation checklists
- **Session Logs** (`docs/sessions/`) - Daily work notes
- **CLAUDE.md** - Real-time session focus (current task right now)
- **Project PRD** (`docs/temporal-cookbook-ui-prd.md`) - Overall vision and phases
