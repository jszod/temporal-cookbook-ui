# PRD Evolution Prompt Template

## When to Use This Prompt

Use this systematic process when implementation sessions reveal that requirements have evolved, been invalidated, or need specificity updates for effective AI-human collaboration.

### PRD Evolution Trigger Conditions

**Run PRD evolution when session involved:**
- Requirements that proved wrong during implementation
- New HIGH SPECIFICITY requirements discovered through technical work
- Performance characteristics that changed significantly
- User needs that emerged through testing or feedback
- Technical constraints that invalidate existing requirements
- Success criteria that need adjustment based on evidence
